# Claude Skills 作成チェックリスト

このチェックリストを使って、スキル作成の各ステップを確認してください。

## 📋 作業開始前

### ユースケース定義
- [ ] 2-3個の具体的なユースケースを定義した
- [ ] 誰がこのスキルを使うか明確にした
- [ ] どのような状況で使うか定義した
- [ ] 達成したい結果を明確にした

### ツール・依存関係の確認
- [ ] 必要なツール（Claude組み込み or MCP）を特定した
- [ ] MCPサーバーを使用する場合、接続可能か確認した
- [ ] 外部スクリプトが必要な場合、実行環境を確認した

### ガイドラインの確認
- [ ] 完全ガイド (PDF) を読んだ
- [ ] 類似のスキル例を確認した
- [ ] ベストプラクティスを理解した

---

## 🏗️ 開発中

### フォルダ構造
- [ ] フォルダ名を kebab-case で命名した (例: `my-skill-name`)
- [ ] スペース、アンダースコア、大文字を使っていない
- [ ] `SKILL.md` ファイルを作成した (大文字小文字正確)
- [ ] 必要に応じて `scripts/`, `references/`, `assets/` フォルダを作成した

### YAML フロントマター
- [ ] `---` で開始・終了している
- [ ] `name` フィールド: kebab-case、スペース・大文字なし
- [ ] `description` フィールド: 「何をするか」と「いつ使うか」を含む
- [ ] description は 1024文字以内
- [ ] XMLタグ (`< >`) を使っていない
- [ ] 具体的なトリガーフレーズを含めた
- [ ] ファイルタイプが関連する場合、明記した

### オプションフィールド (必要に応じて)
- [ ] `license` フィールドを追加した (オープンソースの場合)
- [ ] `compatibility` フィールドで環境要件を記載した
- [ ] `metadata` フィールドで追加情報を記載した
  - author
  - version
  - mcp-server
  - その他のカスタムフィールド

### スキル本体の指示
- [ ] 明確で具体的な指示を書いた
- [ ] 箇条書き・番号付きリストを使用した
- [ ] ステップバイステップの手順を記載した
- [ ] エラーハンドリングの指示を含めた
- [ ] 使用例を提供した
- [ ] トラブルシューティングセクションを追加した

### プログレッシブディスクロージャー
- [ ] SKILL.md は簡潔に保った (5,000語以下推奨)
- [ ] 詳細なドキュメントは `references/` に移動した
- [ ] references/ のファイルへのリンクを明記した

### 禁止事項の確認
- [ ] スキルフォルダ内に `README.md` を含めていない
- [ ] スキル名に `claude` や `anthropic` を使っていない
- [ ] XMLタグを使っていない

---

## 🧪 テスト

### トリガーテスト
- [ ] 明確なクエリでスキルがトリガーされることを確認した
- [ ] パラフレーズされたクエリでもトリガーされることを確認した
- [ ] 無関係なトピックではトリガーされないことを確認した
- [ ] トリガーテストを10-20回実施した

### 機能テスト
- [ ] スキルが正しいアウトプットを生成することを確認した
- [ ] API呼び出しが成功することを確認した (MCP使用時)
- [ ] エラーハンドリングが機能することを確認した
- [ ] エッジケースをテストした

### パフォーマンステスト
- [ ] スキルなしでの実行と比較した
- [ ] ツール呼び出し回数を記録した
- [ ] トークン消費量を確認した
- [ ] 失敗したAPI呼び出しを記録した

### 定性評価
- [ ] ユーザーが次のステップを指示する必要がないか確認した
- [ ] ワークフローが修正なしで完了するか確認した
- [ ] 複数セッションで一貫した結果が得られるか確認した

---

## 📦 アップロード前

### ファイルの最終確認
- [ ] すべてのファイルが正しいフォルダにある
- [ ] 不要なファイル（.DS_Store など）を削除した
- [ ] ファイルパスが正しいか確認した

### 圧縮とアップロード
- [ ] スキルフォルダを .zip ファイルに圧縮した
- [ ] Claude.ai Settings > Capabilities > Skills からアップロードした
  または
- [ ] Claude Code のスキルディレクトリに配置した

---

## 🚀 アップロード後

### 実環境テスト
- [ ] 実際の会話でスキルをテストした
- [ ] 複数の異なるシナリオでテストした
- [ ] ユーザーフィードバックを収集した

### モニタリング
- [ ] アンダートリガー（トリガーされるべき時にされない）を監視
- [ ] オーバートリガー（不要な時にトリガーされる）を監視
- [ ] 実行エラーを監視

### 反復改善
- [ ] フィードバックに基づいて description を改善した
- [ ] 指示をより明確にした
- [ ] エッジケースへの対応を追加した
- [ ] metadata の version フィールドを更新した

---

## 📚 配布（公開する場合）

### ドキュメント整備
- [ ] GitHubリポジトリにREADME.mdを作成した（リポジトリレベル）
- [ ] インストール手順を明記した
- [ ] 使用例とスクリーンショットを追加した
- [ ] ライセンスを明記した

### MCP統合（該当する場合）
- [ ] MCPドキュメントにスキルへのリンクを追加した
- [ ] MCPとスキルを併用する価値を説明した
- [ ] クイックスタートガイドを作成した

---

## 🔧 トラブルシューティング用チェック

### スキルがアップロードできない
- [ ] ファイル名が正確に `SKILL.md` か確認
- [ ] YAMLフロントマターのフォーマットが正しいか確認
- [ ] XMLタグ (`< >`) を使っていないか確認
- [ ] スキル名が kebab-case か確認

### スキルがトリガーされない
- [ ] description が具体的か確認
- [ ] トリガーフレーズが含まれているか確認
- [ ] ファイルタイプが関連する場合、明記しているか確認
- [ ] 他のスキルと競合していないか確認

### スキルがトリガーされすぎる
- [ ] description が具体的すぎないか確認
- [ ] ネガティブトリガー（「使わない場合」）を追加
- [ ] スコープをより限定的にする

### 指示が守られない
- [ ] 指示が簡潔か確認
- [ ] 重要な指示が上部にあるか確認
- [ ] 箇条書きを使っているか確認
- [ ] 曖昧な表現を避けているか確認

### MCP接続エラー
- [ ] MCPサーバーが接続されているか確認
- [ ] API キーが有効か確認
- [ ] ツール名が正しいか確認
- [ ] ツール名が大文字小文字を含めて正確か確認

---

## 💡 追加のヒント

- **skill-creator を活用**: Claude.ai で利用可能な skill-creator スキルを使うと、生成・レビューが簡単
- **小さく始める**: 最初は1つの具体的なタスクに集中し、徐々に拡張する
- **既存スキルを参考に**: [anthropics/skills](https://github.com/anthropics/skills) の公式スキルを参考にする
- **フィードバックを重視**: ユーザーフィードバックに基づいて継続的に改善する
- **バージョン管理**: metadata に version を記載し、変更を追跡する

---

このチェックリストを印刷またはコピーして、スキル作成の際に使用してください。
